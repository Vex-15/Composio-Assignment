Architecture

Agent Pipeline Architecture

The 4-stage research pipeline — from raw app list to verified findings.

1

Composio Catalog Lookup

Query Composio's REST API for apps already in their toolkit catalog. Extract auth schemes, tool counts — free ground truth that demonstrates product understanding.

2

Doc Fetch + LLM Extraction

Async fetch of developer docs with caching + retries. LLM (Gemini) extracts structured fields. Rate-limited, resumable, with per-domain throttling.

3

Low-Confidence Retry

Apps flagged as low confidence get a second pass with different search queries and alternative doc URLs. Catches 15-30% of initial misses.

4

Verification Loop

20-app stratified sample re-researched independently. Field-by-field comparison. Corrections applied to produce verified results.

Where Human Intervention Was Needed

Partner-Gated APIs

Apps like DealCloud, PitchBook, and Amazon Selling Partner require partnership agreements. The agent correctly identified "partner_gated" but couldn't verify specific API capabilities behind the gate. A human confirmed the access model by reading contact-sales pages.

JavaScript-Rendered Doc Sites

Several developer portals (e.g., Squarespace, BigCommerce) serve docs via JavaScript SPAs. The HTTP fetch returned empty/minimal content. Human verified by opening in browser and cross-checking auth details manually.

Ambiguous Auth Methods

Some apps list multiple auth methods without clear hierarchy (e.g., Zoho's OAuth + API key + token). The LLM sometimes picked only one. Human review ensured all methods were captured.

Lessons Learned

What the pipeline got wrong, why, and how to improve.

LLM Extraction Isn't Perfect

The first-pass LLM extraction averaged ~75-85% field-level accuracy. Main failure mode: confusing API key vs OAuth when docs mention both. Solution: require the LLM to list ALL auth methods, not pick one.

JavaScript SPAs Break Fetching

~10-15 apps serve docs via JS-heavy SPAs that return minimal HTML to a simple HTTP fetch. These apps systematically got "unknown" on first pass. A headless browser would catch them.

Pricing Pages Are Hard

Determining "self_serve_free" vs "self_serve_trial" vs "paid_plan_required" was the lowest-accuracy field. Pricing pages use marketing language that LLMs struggle with.

Composio Catalog = Best Source

For apps in Composio's catalog, auth_methods accuracy was near 100%. Using the Composio SDK as ground truth was the single best accuracy improvement in the pipeline.

MCP Is Still Emerging

Very few apps have official MCP servers. The LLM sometimes hallucinated MCP support when it found any AI/agent integration. Strict evidence-URL requirements caught most false positives.

Verification Catches Real Errors

The verification loop wasn't theater — it found genuine disagreements on ~20-30% of sampled apps. Most corrections were subtle (e.g., missing a secondary auth method, wrong access model tier).
